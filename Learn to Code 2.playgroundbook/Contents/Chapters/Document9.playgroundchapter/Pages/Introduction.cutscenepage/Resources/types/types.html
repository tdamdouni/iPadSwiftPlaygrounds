In dot notation, the part before the dot references the instance (your specific house).
Here, you’re changing the isActive property of the bluePortal instance to false, which deactivates the portal.
In programming, a type is like a blueprint. An instance is like a house that you build from the blueprint.
First you refer to that house by name, and then you tell it what to do.
The part after the dot calls the method, openGarageDoor(), from the myHouse instance.
